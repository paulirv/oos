# Prior Art Analysis

Specifications, standards, and initiatives relevant to OpenOrigin.

## Content Provenance & AI Transparency

| Initiative | Type | Relevance to OOS |
|------------|------|-------------------|
| [C2PA / Content Credentials](c2pa.md) | Technical standard | Cryptographic media provenance — complementary, not competing |
| [EU AI Act](eu-ai-act.md) | Regulation | Mandatory AI labeling — OOS exceeds compliance minimums |
| Platform labels (YouTube, Meta, TikTok) | Proprietary | Inconsistent, platform-specific — OOS provides a universal standard |
| OpenAI Model Spec | Provider spec | Provider-side behavior — OOS is publisher-side declaration |

## Structural Models

| Specification | What OOS borrows |
|---------------|------------------|
| [WCAG 2.x](wcag-structure.md) | Principles > Guidelines > Success Criteria > Techniques |
| [Creative Commons](creative-commons.md) | Three-layer approach: human-readable, machine-readable, legal |
| [Schema.org](schema-org.md) | Machine-readable metadata vocabulary, adoption strategy |
| OpenAPI Specification | Community-driven governance model |

## Machine-Readable Metadata

| Standard | Relationship |
|----------|-------------|
| Schema.org | OOS builds on/extends schema.org types where possible |
| Dublin Core | Precedent for content metadata in HTML |
| Open Graph Protocol | Model for simple meta tag adoption |
| robots.txt / meta robots | Model for machine-readable content directives |
