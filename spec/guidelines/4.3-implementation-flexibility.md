# Guideline 4.3: Implementation Flexibility

**Principle:** [Practical](../principles/practical.md)

Multiple valid implementation paths exist.

## Intent

Publishers choose the format that fits their stack. A WordPress site might use meta tags. An API might use HTTP headers. A static site might use frontmatter. All are equally valid paths to conformance.

## Success Criteria

| ID | Level | Criterion |
|----|-------|-----------|
| 4.3.1 | L1 | Any recognized format achieves conformance |
