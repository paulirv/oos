# Guideline 4.3: Implementation Flexibility

**Principle:** [Practical](../principles/practical.md)

Multiple valid implementation paths exist.

## Intent

Publishers choose the format that fits their stack. A WordPress site might use meta tags. An API might use HTTP headers. A static site might use frontmatter. All are equally valid paths to conformance.

## Success Criteria

This guideline has no per-content success criteria. Format neutrality is a design property of the specification itself. See [Conformance: Design Properties](../conformance.md#design-properties).

## Techniques

- [T-HTML-META](../techniques/html-meta-tag.md) — For HTML pages
- [T-HTTP-HEADER](../techniques/http-header.md) — For API responses
- [T-FRONTMATTER](../techniques/frontmatter.md) — For static content
- [T-JSON-LD](../techniques/json-ld.md) — For rich structured data
