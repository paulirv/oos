# Technique: Frontmatter

**ID:** T-FRONTMATTER
**Type:** Sufficient for L1-L2, Source for L3
**Applies to:** Static content (Markdown, MDX, YAML-based content)

## Description

YAML frontmatter in content files provides origin declarations at the source level. Static site generators can transform frontmatter into HTML meta tags or JSON-LD at build time.

## L1 Implementation

```yaml
---
openorigin: ai-assisted
---
```

## L2 Implementation

```yaml
---
openorigin: ai-assisted
openorigin_detail:
  conformance: L2
  author:
    name: Jane Smith
    role: primary-author
  tools:
    - name: Claude
      provider: Anthropic
      role: editing
  date: 2026-02-10
---
```

## SSG Integration

Frontmatter is a source format. The output format (meta tags, JSON-LD) is generated by the static site generator. Example template:

```html
<!-- Hugo/Eleventy/Astro partial -->
<meta name="openorigin" content="{{ openorigin }}">
{% if openorigin_detail %}
<script type="application/ld+json">
  {{ openorigin_detail | jsonld }}
</script>
{% endif %}
```

## Benefits

- Natural for static sites and Markdown-based content
- Human-readable in source files
- Build-time transformation to any output format
- Version-controlled alongside content
